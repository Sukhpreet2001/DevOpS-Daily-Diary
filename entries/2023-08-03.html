Verifying your GitHub Pages

Prerequisites:

  • An already set up GitHub Pages website.
  • A Google Webmaster account.

Steps:

1. Obtain the Verification Code:

  1. In your Google Webmaster account, navigate to the property (website) you wish to verify.
  2. Choose the "HTML tag" verification method.
  3. Copy the verification code provided by Google.

2. Open Your GitHub Repository:

  1. Log in to your GitHub account.
  2. Open the repository associated with your GitHub Pages website.

3. Edit the HTML File:

  1. In your repository, locate the HTML file where you want to add the verification code.
  2. This is typically the main HTML file that represents your website (e.g., index.html).

4. Insert the Verification Code:

  1. Open the HTML file in edit mode.
  2. Locate the <head> section of your HTML code.
  3. Paste the verification code provided by Google inside the <head> section.
  4. Save your changes.

5. Commit and Push Changes:

  1. Open GitHub Desktop or your preferred Git client.
  2. Select the repository containing your GitHub Pages website.
  3. You should see the updated HTML file listed as a changed file.
  4. Write a meaningful commit message, such as "Adding Google Webmaster verification code."
  5. Commit the changes to the "main" branch (or the appropriate branch if different).
  6. Push the committed changes to the remote repository on GitHub.

6. Verify with Google Webmaster:

  1. Return to your Google Webmaster account.
  2. In the verification settings, click "Verify."
  3. Google will check your website's HTML for the verification code.
  4. Upon successful verification, you'll receive a confirmation from Google.

7. Access Your Updated Website:

  1. Give a moment for the verification to take effect.
  2. Visit your Google Webmaster account to confirm the successful verification.
