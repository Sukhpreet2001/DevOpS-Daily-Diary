Linux with Docker

Linux - Why Do We Use It?

Linux is a widely used operating system known for its versatility and reliability. There are several advantages to using Linux:

1. Open Source and Customizability

Linux is an open-source operating system, which means its source code is freely available. This allows users to customize and modify the OS according to their needs, providing greater flexibility and control over the system.

2. Stability and Reliability

Linux is highly regarded for its stability and reliability, making it a preferred choice for servers and critical infrastructure. It has a robust memory management system and is less prone to crashes and system failures.

3. Security

Linux is designed with security in mind. It features a strong user permission model and receives regular security updates and patches to promptly address vulnerabilities.

4. Performance

Linux typically has lower resource requirements compared to other operating systems, allowing it to run efficiently on various hardware configurations.

5. Wide Range of Applications

Linux finds application in a wide range of scenarios, from web servers and cloud infrastructure to embedded systems and IoT devices.

In practice, Linux and Docker are often used together, taking advantage of both technologies. Docker runs natively on Linux, and many Docker images are built on top of Linux-based distributions. Developers use Linux as the underlying operating system on servers, and then Docker containers are used to package and deploy applications and services in a consistent and reproducible manner.

By combining the stability, performance, and customizability of Linux with the portability, isolation, and scalability of Docker, developers can create a powerful and flexible infrastructure to deploy and manage applications effectively. Docker complements Linux, enabling efficient resource utilization and simplifying the deployment and management of complex applications in various environments.

Commands for Linux (Ubuntu)

Below are some commonly used commands in Linux (Ubuntu) terminal:

  1. Open hub.docker.com - Copy the command and paste it in the terminal (play-with-docker)
  2. docker run -it ubuntu
  3. echo "name" (with root @id)
  4. history (display command history)
  5. ls (list directories)
  6. pwd (display present working directory)
  7. cd lib (change to the 'lib' directory)
  8. cd / (move to the root directory)
  9. apt list (list all packages)
  10. apt list update
  11. apt install yarn
  12. exit

In Linux, directories are represented using '/', while in Windows, they are represented using '\'. The command line usage and syntax may vary between the two operating systems.
